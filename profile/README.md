# ✈️  SingleFlight Lab

> 하나의 지식으로, 함께 더 멀리

더 높은 곳으로 날아오르기 위해서 백엔드, 프론트, 인프라, 클라우드 등 다양한 기술적 역량 향상을 위한 연구실입니다.


## About

SingleFlight는 캐시 스탬피드를 방지하기 위해 동일한 요청을 하나로 병합하는 백엔드 패턴에서 영감을 받았습니다.

단순히 QnA 방식의 딱딱한 스터디가 아닌, 한 사람이 학습한 내용을 여러 사람과 공유하고, 한 답변에 대해서 어떤 질문이 나올 수 있을까 의논하며 함께 성장하는 문화를 만들어가려고 합니다.

## 🛠 How We Study

### 1. 발표 기반 학습

매주 한 명이 특정 주제를 발표합니다.

단순 개념 설명이 아니라:

- 왜 사용하는지
- 어떤 문제가 발생하는지
- 실무에서는 어떻게 해결하는지

까지 함께 다룹니다.

---

### 2. 기술 면접 대비

실제 면접처럼 질문하고 답변합니다.

예시:

- Redis와 Caffeine 차이는?
- 왜 Kafka를 사용하는가?
- CAS와 synchronized의 차이점은?
- 싱글 플라이트 패턴이란?

---

### 3. 트러블슈팅 공유

직접 겪은 문제를 함께 분석합니다.

```text
예:
- Cache Stampede
- N+1 문제
- DeadLock
- GC 이슈
- Connection Pool 고갈
```

## 🛫 Why "SingleFlight"?

SingleFlight는 동일한 요청을 하나로 병합하여 불필요한 중복 작업을 줄이는 패턴입니다.

우리도 마찬가지로,

- 각자의 학습 경험을 공유하고
- 서로의 시행착오를 줄이며
- 더 빠르게 성장하고자 합니다.
